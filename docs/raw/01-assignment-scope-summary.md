# Raw Notes: Assignment Scope Summary

Source: `Movie Ticket Booking System.pdf` (SDE-2 Take-Home)

## Product Requirements

- Multi-city, multi-theater, multi-show, seat-level booking
- Time-bound seat holds with auto-release on expiry
- Pricing tiers: regular, premium, weekend + discount codes
- Payment, booking confirmation, configurable refund policies
- Concurrency: no double seat allocation
- Async notifications (confirmation + reminders)
- Roles: ADMIN, CUSTOMER

## Mandatory Deliverables

- REST APIs, database persistence, RBAC, validation, unit + integration tests
- GitHub repo with multiple commits, README.md, AGENTS.md, skills, raw files
- Loom video (max 10 min)

## Explicitly Out of Scope

- UI/frontend, deployment/CI/CD, microservices, OAuth/SSO/MFA, production observability

## Agent Interpretation

Requirements are intentionally open-ended. Scoping decisions (entities, APIs, edge cases) are part of the evaluation. Interpret generously — breadth and depth both matter.
